# diff-graphcodeBERT 接口设计文档

> 基础地址：`http://localhost:8000`
> 接口文档：`http://localhost:8000/docs`（Swagger UI）

---

## 接口总览

| 接口 | 方法 | 说明 | 需要模型 |
|------|------|------|---------|
| `/health` | GET | 健康检查 | 否 |
| `/model-status` | GET | 模型加载状态 | 否 |
| `/compare` | POST | 两段代码的语义相似度比较 | 是 |
| `/ast-similarity` | POST | 基于 AST 编码的结构相似度比较 | 是 |
| `/semantic-diff` | POST | 基于语义的代码差异分析 | 是 |
| `/analyze` | POST | 分析单段代码的结构信息 | 是 |
| `/batch-compare` | POST | 批量比较代码相似度 | 是 |
| `/preload` | POST | 预加载模型 | 是 |

---

## 1. 健康检查

**GET** `/health`

检查服务运行状态和模型加载情况。

**响应示例**

```json
{
  "status": "healthy",
  "model_loaded": true,
  "model_loading": false,
  "model_error": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| status | string | 服务状态，固定为 `"healthy"` |
| model_loaded | boolean | 模型是否已加载完成 |
| model_loading | boolean | 模型是否正在加载中 |
| model_error | string/null | 模型加载失败的错误信息 |

---

## 2. 模型加载状态

**GET** `/model-status`

获取 GraphCodeBERT 模型的加载进度。

**响应示例**

```json
{
  "status": "ready",
  "message": "模型已就绪，可以开始使用"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| status | string | `ready` / `loading` / `error` / `not_started` |
| message | string | 可读的状态描述 |

**status 枚举值**

| 值 | 说明 |
|----|------|
| `ready` | 模型已就绪，可以调用比较接口 |
| `loading` | 模型正在加载中（首次启动需下载约 500MB） |
| `error` | 模型加载失败 |
| `not_started` | 模型尚未开始加载 |

## 3. 代码语义相似度比较（核心接口）

**POST** `/compare`

使用 GraphCodeBERT 比较两段代码的语义相似度。内部流程：去注释 → tree-sitter 解析 → DFG 提取 → 模型编码 → 校准余弦相似度。

### 请求

```json
{
  "code1": "function add(a, b) { return a + b; }",
  "code2": "const sum = (x, y) => x + y;",
  "lang": "javascript"
}
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| code1 | string | 是 | - | 第一段代码 |
| code2 | string | 是 | - | 第二段代码 |
| lang | string | 否 | `"c"` | 编程语言（python/c/java/javascript/go/ruby） |

### 响应

```json
{
  "similarity": 0.82,
  "similarity_percent": 82.0,
  "code1_analysis": {
    "token_count": 12,
    "dfg_edges": 3
  },
  "code2_analysis": {
    "token_count": 10,
    "dfg_edges": 2
  },
  "interpretation": "较高相似度：代码存在明显的相似特征，可能使用了相似的算法或模式"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| similarity | float | 相似度分数（0-1，1 为完全相同） |
| similarity_percent | float | 相似度百分比（0-100） |
| code1_analysis | object | 代码1 的分析结果 |
| code1_analysis.token_count | int | 代码1 的 token 数量 |
| code1_analysis.dfg_edges | int | 代码1 的 DFG 边数量 |
| code2_analysis | object | 代码2 的分析结果（结构同上） |
| interpretation | string | 相似度的可读解释文本 |

**interpretation 分级标准**

| 相似度范围 | 解释 |
|-----------|------|
| >= 0.95 | 极高相似度：两段代码几乎相同 |
| >= 0.85 | 高相似度：功能和结构高度相似 |
| >= 0.70 | 较高相似度：存在明显的相似特征 |
| >= 0.50 | 中等相似度：有部分相似，也有显著差异 |
| >= 0.30 | 较低相似度：少量相似特征，整体差异大 |
| < 0.30 | 低相似度：差异明显，功能或结构不同 |

## 4. AST 结构相似度比较

**POST** `/ast-similarity`

基于 AST 的 SBT 序列化字符串计算结构相似度。前端将代码 AST 序列化为 SBT 格式后发送到此接口，由 GraphCodeBERT 编码为向量并计算余弦相似度。

### 请求

```json
{
  "sbt1": "( Program ( FunctionDeclaration[params:2] ( _ID_ ) ( BlockStatement ( ReturnStatement ( BinaryExpression[+] ( _ID_ ) ( _ID_ ) ) ) ) ) )",
  "sbt2": "( Program ( VariableDeclaration[const] ( VariableDeclarator ( _ID_ ) ( ArrowFunctionExpression[params:2] ( _ID_ ) ( BinaryExpression[+] ( _ID_ ) ( _ID_ ) ) ) ) ) )"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| sbt1 | string | 是 | 第一份代码的 AST SBT 序列化字符串 |
| sbt2 | string | 是 | 第二份代码的 AST SBT 序列化字符串 |

### 响应

```json
{
  "similarity": 0.91,
  "similarity_percent": 91.0,
  "interpretation": "AST 结构非常相似，仅有少量结构差异",
  "sbt1_tokens": 28,
  "sbt2_tokens": 32
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| similarity | float | 结构相似度（0-1） |
| similarity_percent | float | 相似度百分比（0-100） |
| interpretation | string | 相似度的可读解释 |
| sbt1_tokens | int | SBT1 的 token 数量 |
| sbt2_tokens | int | SBT2 的 token 数量 |

**interpretation 分级标准**

| 相似度范围 | 解释 |
|-----------|------|
| >= 0.95 | AST 结构几乎完全相同 |
| >= 0.85 | AST 结构非常相似，仅有少量差异 |
| >= 0.70 | AST 结构相似，存在一些不同 |
| >= 0.50 | 有一定相似性，但有明显差异 |
| >= 0.30 | 差异较大，可能是不同实现方式 |
| < 0.30 | 差异很大，结构完全不同 |

## 5. 语义差异分析

**POST** `/semantic-diff`

基于语义的代码差异分析，是本系统最核心的接口。与传统 diff 的区别：能区分"语义等价变更"（格式、样式、等价写法）和"真实差异"。

### 请求

```json
{
  "code1": "function add(a, b) {\n  return a + b;\n}",
  "code2": "const add = (x, y) => {\n  return x + y;\n}",
  "lang": "javascript",
  "equivalent_threshold": 0.85
}
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| code1 | string | 是 | - | 原始代码 |
| code2 | string | 是 | - | 待比较代码 |
| lang | string | 否 | `"c"` | 编程语言 |
| equivalent_threshold | float | 否 | `0.85` | 语义等价阈值（0-1） |

### 响应

```json
{
  "similarity": 0.78,
  "stats": {
    "overall_similarity": 0.78,
    "total_lines1": 3,
    "total_lines2": 3,
    "exact_matches": 0,
    "semantic_equivalent": 2,
    "different_lines": 1,
    "removed_lines": 0,
    "added_lines": 0
  },
  "alignments": [
    {
      "line1": 1,
      "line2": 1,
      "content1": "function add(a, b) {",
      "content2": "const add = (x, y) => {",
      "similarity": 0.87,
      "type": "equivalent",
      "is_semantic_equivalent": true
    },
    {
      "line1": 2,
      "line2": 2,
      "content1": "  return a + b;",
      "content2": "  return x + y;",
      "similarity": 0.92,
      "type": "equivalent",
      "is_semantic_equivalent": true
    },
    {
      "line1": 3,
      "line2": 3,
      "content1": "}",
      "content2": "}",
      "similarity": 1.0,
      "type": "exact",
      "is_semantic_equivalent": true
    }
  ],
  "removed": [],
  "added": [],
  "interpretation": "两段代码功能相似（相似度 78.0%）。语义等价变更 2 行（如变量重命名、顺序调整等，无需关注）。"
}
```

### 响应字段说明

**顶层字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| similarity | float | 整体语义相似度（0-1） |
| stats | DiffStats | 差异统计信息 |
| alignments | LineAlignment[] | 行对齐列表 |
| removed | UnmatchedLine[] | 被删除的行（代码1有、代码2没有） |
| added | UnmatchedLine[] | 新增的行（代码2有、代码1没有） |
| interpretation | string | 可读的差异分析结论 |

**DiffStats 字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| overall_similarity | float | 整体相似度 |
| total_lines1 | int | 代码1 非空行数 |
| total_lines2 | int | 代码2 非空行数 |
| exact_matches | int | 完全相同的行数 |
| semantic_equivalent | int | 语义等价的行数（格式/样式/模型判定） |
| different_lines | int | 存在差异的行数 |
| removed_lines | int | 被删除的行数 |
| added_lines | int | 新增的行数 |

**LineAlignment 字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| line1 | int | 代码1 的行号 |
| line2 | int | 代码2 的行号 |
| content1 | string | 代码1 的行内容 |
| content2 | string | 代码2 的行内容 |
| similarity | float | 该行的相似度（0-1） |
| type | string | 对齐类型：`exact` / `equivalent` / `different` |
| is_semantic_equivalent | bool | 是否语义等价 |

**type 枚举值**

| 值 | 说明 | 前端展示 |
|----|------|---------|
| `exact` | 完全相同 | 不高亮 |
| `equivalent` | 语义等价（格式/样式/模型判定） | 灰色高亮 |
| `different` | 存在真实差异 | 红色高亮 |

**UnmatchedLine 字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| line | int | 行号 |
| content | string | 行内容 |

## 6. 代码结构分析

**POST** `/analyze`

分析单段代码的结构信息，返回预处理结果和 DFG 信息。

### 请求

```json
{
  "code": "int add(int a, int b) { return a + b; }",
  "lang": "c"
}
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| code | string | 是 | - | 代码内容 |
| lang | string | 否 | `"c"` | 编程语言 |

### 响应

```json
{
  "cleaned_code": "int add(int a, int b) { return a + b; }",
  "token_count": 15,
  "dfg_edges": 3,
  "dfg_string": "a:comesFrom:0 b:comesFrom:1 add:computedFrom:0,1"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| cleaned_code | string | 去注释后的代码 |
| token_count | int | token 数量 |
| dfg_edges | int | DFG 数据流图的边数量 |
| dfg_string | string | DFG 的字符串表示 |

## 7. 批量代码比较

**POST** `/batch-compare`

一段查询代码与多段代码批量比较，返回按相似度降序排列的结果。

### 请求

```json
{
  "query_code": "function add(a, b) { return a + b; }",
  "code_list": [
    "function sub(a, b) { return a - b; }",
    "const sum = (x, y) => x + y;",
    "console.log('hello world');"
  ],
  "lang": "javascript"
}
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| query_code | string | 是 | - | 查询代码 |
| code_list | string[] | 是 | - | 待比较的代码列表 |
| lang | string | 否 | `"c"` | 编程语言 |

### 响应

```json
{
  "results": [
    {
      "index": 1,
      "similarity": 0.89,
      "code_preview": "const sum = (x, y) => x + y;"
    },
    {
      "index": 0,
      "similarity": 0.75,
      "code_preview": "function sub(a, b) { return a - b; }"
    },
    {
      "index": 2,
      "similarity": 0.12,
      "code_preview": "console.log('hello world');"
    }
  ]
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| results | BatchCompareResult[] | 结果列表（按相似度降序排列） |
| results[].index | int | 在 code_list 中的原始索引 |
| results[].similarity | float | 相似度分数 |
| results[].code_preview | string | 代码预览（最多 100 字符） |

## 8. 预加载模型

**POST** `/preload`

手动触发模型加载（正常情况下服务启动时自动后台加载）。

### 响应

```json
{
  "status": "success",
  "message": "模型加载完成"
}
```

---

## 错误响应

所有接口在失败时返回统一格式：

**模型未就绪（503）**

```json
{
  "detail": "模型正在加载中，请稍后再试。首次启动需要下载约500MB的模型文件。"
}
```

**服务器错误（500）**

```json
{
  "detail": "错误描述信息"
}
```

## 前端调用示例

```typescript
// 1. 检查模型状态
const status = await fetch('/api/model-status').then(r => r.json());

// 2. AST 向量相似度（前端先序列化 SBT，再发到后端编码）
const astRes = await fetch('/api/ast-similarity', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ sbt1, sbt2 }),
}).then(r => r.json());

// 3. GraphCodeBERT 语义相似度
const cmpRes = await fetch('/api/compare', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ code1, code2, lang: 'javascript' }),
}).then(r => r.json());

// 4. 语义 Diff
const diffRes = await fetch('/api/semantic-diff', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ code1, code2, lang: 'javascript' }),
}).then(r => r.json());
```

> 注意：前端通过 Vite 代理 `/api/*` → `http://localhost:8000/*`
