# 代码相似度比对最佳方案设计

## 目标

比较**样板代码**和**AI生成代码**，找出结构和功能上的差异：
- 顺序不同 → 不算差异
- 变量名不同 → 不算差异
- 样式值不同 → 不算差异
- 函数实现方式不同但功能等价 → 不算差异

---

## 推荐方案：基于 AST 的功能单元匹配

## 详细设计

### 1. 功能单元定义

| 功能单元类型 | 说明 | 示例 |
|-------------|------|------|
| **Import 单元** | 导入语句 | `import { useState } from 'react'` |
| **函数单元** | 函数/箭头函数 | `function foo() {}` 或 `const foo = () => {}` |
| **组件单元** | React 组件 | `const MyComponent = () => <div>...</div>` |
| **类单元** | 类定义 | `class Foo extends Bar {}` |
| **Hook 单元** | React Hook 调用 | `const [x, setX] = useState()` |
| **副作用单元** | useEffect 等 | `useEffect(() => {...}, [deps])` |

### 2. 规范化签名生成

#### 2.1 函数签名规范化

```javascript
// 原始代码 A
function calculateTotal(items, discount) {
  let sum = 0;
  for (const item of items) {
    sum += item.price;
  }
  return sum * (1 - discount);
}

// 原始代码 B
const calcSum = (products, rate) => {
  return products.reduce((acc, p) => acc + p.price, 0) * (1 - rate);
};

// 规范化签名（两者相同）
{
  type: 'Function',
  params: 2,
  hasLoop: true,        // for...of 和 reduce 都算循环
  hasReturn: true,
  bodyStructure: 'Loop -> BinaryExpression -> Return'
}
```

#### 2.2 组件签名规范化

```javascript
// 原始代码 A
const FormModal = ({ visible, onClose }) => {
  const [loading, setLoading] = useState(false);
  return (
    <Modal visible={visible} onCancel={onClose}>
      <Form>
        <Input name="title" />
        <Button loading={loading}>Submit</Button>
      </Form>
    </Modal>
  );
};

// 原始代码 B  
function MyFormDialog({ show, handleClose }) {
  const [isLoading, setIsLoading] = useState(false);
  return (
    <Modal visible={show} onCancel={handleClose}>
      <Form>
        <Input name="title" />
        <Button loading={isLoading}>Submit</Button>
      </Form>
    </Modal>
  );
}

// 规范化签名（两者相同）
{
  type: 'Component',
  props: ['_BOOL_', '_FUNC_'],  // 忽略具体名称，只保留类型模式
  hooks: ['useState'],
  jsx: {
    root: 'Modal',
    children: ['Form'],
    descendants: ['Input', 'Button']
  }
}
```

### 3. 集合匹配算法

```typescript
interface MatchResult {
  matched: FunctionUnit[];      // 完全匹配的功能单元
  similar: SimilarPair[];       // 相似但有差异的功能单元对
  onlyInA: FunctionUnit[];      // 只在样板代码中存在
  onlyInB: FunctionUnit[];      // 只在 AI 代码中存在
}

function matchFunctionUnits(unitsA: FunctionUnit[], unitsB: FunctionUnit[]): MatchResult {
  const result: MatchResult = { matched: [], similar: [], onlyInA: [], onlyInB: [] };
  
  const unmatchedB = new Set(unitsB);
  
  for (const unitA of unitsA) {
    // 1. 先找完全匹配（签名相同）
    const exactMatch = [...unmatchedB].find(b => 
      b.signature === unitA.signature
    );
    
    if (exactMatch) {
      result.matched.push(unitA);
      unmatchedB.delete(exactMatch);
      continue;
    }
    
    // 2. 再找相似匹配（签名相似度 > 0.8）
    const similarMatch = [...unmatchedB].find(b => 
      signatureSimilarity(unitA.signature, b.signature) > 0.8
    );
    
    if (similarMatch) {
      result.similar.push({ a: unitA, b: similarMatch });
      unmatchedB.delete(similarMatch);
      continue;
    }
    
    // 3. 没有匹配
    result.onlyInA.push(unitA);
  }
  
  result.onlyInB = [...unmatchedB];
  return result;
}
```

## 实现步骤

### Phase 1：功能单元提取器

```typescript
class FunctionUnitExtractor {
  extract(code: string): FunctionUnit[] {
    const ast = parse(code, { plugins: ['jsx', 'typescript'] });
    const units: FunctionUnit[] = [];
    
    traverse(ast, {
      // 提取 Import
      ImportDeclaration(path) {
        units.push(createImportUnit(path.node));
      },
      
      // 提取函数/组件
      'FunctionDeclaration|ArrowFunctionExpression'(path) {
        if (isReactComponent(path)) {
          units.push(createComponentUnit(path));
        } else {
          units.push(createFunctionUnit(path));
        }
      },
      
      // 提取 Hook 调用
      CallExpression(path) {
        if (isHookCall(path)) {
          units.push(createHookUnit(path));
        }
      }
    });
    
    return units;
  }
}
```

### Phase 2：签名生成器

```typescript
class SignatureGenerator {
  generateFunctionSignature(unit: FunctionUnit): string {
    return JSON.stringify({
      type: 'Function',
      params: unit.params.length,
      paramTypes: unit.params.map(p => this.inferType(p)),
      bodyHash: this.hashBody(unit.body),
      controlFlow: this.extractControlFlow(unit.body),
      calls: this.extractCalls(unit.body).sort(), // 排序使顺序无关
    });
  }
  
  generateComponentSignature(unit: ComponentUnit): string {
    return JSON.stringify({
      type: 'Component',
      propsPattern: this.normalizeProps(unit.props),
      hooks: unit.hooks.map(h => h.type).sort(),
      jsxStructure: this.normalizeJSX(unit.jsx),
    });
  }
  
  // 规范化 JSX 结构（忽略属性值，保留结构）
  normalizeJSX(jsx: JSXElement): object {
    return {
      tag: this.normalizeTag(jsx.tagName),  // div/<>/Fragment → _CONTAINER_
      structuralAttrs: jsx.attributes
        .filter(a => !isStyleAttribute(a))
        .map(a => a.name)
        .sort(),
      children: jsx.children
        .filter(c => c.type === 'JSXElement')
        .map(c => this.normalizeJSX(c))
        .sort((a, b) => JSON.stringify(a).localeCompare(JSON.stringify(b))),
    };
  }
}
```

### Phase 3：匹配器

```typescript
class FunctionUnitMatcher {
  match(unitsA: FunctionUnit[], unitsB: FunctionUnit[]): MatchResult {
    // 按类型分组
    const groupA = this.groupByType(unitsA);
    const groupB = this.groupByType(unitsB);
    
    const result: MatchResult = { matched: [], similar: [], onlyInA: [], onlyInB: [] };
    
    // 对每种类型分别匹配
    for (const type of ['Import', 'Component', 'Function', 'Hook']) {
      const typeResult = this.matchGroup(
        groupA.get(type) || [],
        groupB.get(type) || []
      );
      
      result.matched.push(...typeResult.matched);
      result.similar.push(...typeResult.similar);
      result.onlyInA.push(...typeResult.onlyInA);
      result.onlyInB.push(...typeResult.onlyInB);
    }
    
    return result;
  }
}
```

## 与当前实现的对比

| 特性 | 当前实现 | 新方案 |
|-----|---------|--------|
| 比较粒度 | 行级 + 节点计数 | 功能单元级 |
| 顺序敏感 | 是（行级 diff） | 否（集合匹配） |
| 变量名敏感 | 部分处理 | 完全忽略 |
| 可解释性 | 行差异 | 功能差异 |
| 匹配精度 | 中等 | 高 |
