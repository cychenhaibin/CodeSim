## 1. 概述

### 1.1 什么是代码相似度

代码相似度是衡量两段代码在某种度量标准下的相似程度。根据不同的应用场景，"相似"可以有不同的含义：

| 相似类型 | 描述 | 示例 |
|---------|------|------|
| **文本相似** | 字符/行级别的文本匹配 | 完全复制的代码 |
| **结构相似** | 语法结构相同，变量名可能不同 | 变量重命名后的代码 |
| **语义相似** | 功能等价，实现方式可能不同 | for 循环 vs map() |

### 1.2 相似度检测的层次

```
┌─────────────────────────────────────────────────────┐
│                   语义层 (Semantic)                   │
│    理解代码的功能和意图，最接近人类的理解方式            │
├─────────────────────────────────────────────────────┤
│                   语法层 (Syntactic)                  │
│    分析代码的语法结构（AST），忽略格式差异              │
├─────────────────────────────────────────────────────┤
│                   词法层 (Lexical)                    │
│    将代码分解为 Token 序列进行比较                     │
├─────────────────────────────────────────────────────┤
│                   文本层 (Textual)                    │
│    直接比较源代码的字符串内容                          │
└─────────────────────────────────────────────────────┘
```

### 1.3 代码克隆的四种类型

| 类型 | 名称 | 描述 | 检测难度 |
|-----|------|------|---------|
| Type-1 | 完全克隆 | 除空白和注释外完全相同 | 简单 |
| Type-2 | 重命名克隆 | Type-1 + 标识符/字面量重命名 | 中等 |
| Type-3 | 近似克隆 | Type-2 + 语句增删改 | 较难 |
| Type-4 | 语义克隆 | 功能相同但实现不同 | 困难 |

## 2. 基于文本的方法

### 2.1 核心思想

将代码视为纯文本字符串，使用字符串匹配算法计算相似度。

### 2.2 常用算法

#### 2.2.1 编辑距离（Levenshtein Distance）

计算将一个字符串转换为另一个所需的最少编辑操作数。

```python
def levenshtein_distance(s1, s2):
    """
    编辑距离：插入、删除、替换操作的最小次数
    """
    m, n = len(s1), len(s2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    for i in range(m + 1):
        dp[i][0] = i
    for j in range(n + 1):
        dp[0][j] = j
    
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1]
            else:
                dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])
    
    return dp[m][n]

# 相似度 = 1 - (编辑距离 / max(len(s1), len(s2)))
```

#### 2.2.2 最长公共子序列（LCS）

```python
def lcs_similarity(s1, s2):
    """
    基于最长公共子序列的相似度
    """
    m, n = len(s1), len(s2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    
    lcs_length = dp[m][n]
    return 2 * lcs_length / (m + n)  # Dice 系数
```

#### 2.2.3 行级 Diff（difflib）

Python 的 `difflib` 库使用 Ratcliff/Obershelp 算法：

```python
from difflib import SequenceMatcher, unified_diff

def line_similarity(code1, code2):
    """
    行级相似度比较
    """
    lines1 = code1.splitlines()
    lines2 = code2.splitlines()
    
    matcher = SequenceMatcher(None, lines1, lines2)
    return matcher.ratio()  # 0.0 - 1.0
```

### 2.3 优缺点

| 优点 | 缺点 |
|-----|------|
| 实现简单 | 对格式敏感（空格、换行） |
| 计算快速 | 无法识别语义等价 |
| 无需解析 | 变量重命名即失效 |

### 2.4 适用场景

- 快速初筛
- 检测完全复制的代码
- 版本差异对比

## 3. 基于词法的方法（Token）

### 3.1 核心思想

将代码转换为 Token（词法单元）序列，忽略空白和注释，然后比较 Token 序列。

### 3.2 Token 化过程

```
源代码:
const sum = (a, b) => a + b;

Token 序列:
[CONST, IDENTIFIER, EQUALS, LPAREN, IDENTIFIER, COMMA, 
 IDENTIFIER, RPAREN, ARROW, IDENTIFIER, PLUS, IDENTIFIER, SEMICOLON]

归一化后:
[CONST, ID, EQUALS, LPAREN, ID, COMMA, ID, RPAREN, ARROW, ID, PLUS, ID, SEMICOLON]
```

### 3.3 关键技术

#### 3.3.1 Token 归一化

```python
def normalize_tokens(tokens):
    """
    归一化 Token：
    - 所有标识符 → ID
    - 所有字符串 → STR
    - 所有数字 → NUM
    """
    normalized = []
    for token in tokens:
        if token.type == 'IDENTIFIER':
            normalized.append('ID')
        elif token.type == 'STRING':
            normalized.append('STR')
        elif token.type == 'NUMBER':
            normalized.append('NUM')
        else:
            normalized.append(token.type)
    return normalized
```

#### 3.3.2 N-gram 指纹

```python
def generate_ngrams(tokens, n=5):
    """
    生成 n-gram 指纹用于快速匹配
    """
    ngrams = set()
    for i in range(len(tokens) - n + 1):
        gram = tuple(tokens[i:i+n])
        ngrams.add(hash(gram))
    return ngrams

def jaccard_similarity(set1, set2):
    """
    Jaccard 相似度
    """
    intersection = len(set1 & set2)
    union = len(set1 | set2)
    return intersection / union if union > 0 else 0
```

### 3.4 经典工具：JPlag / MOSS

这些工具的核心算法：

1. **Token 化**：将代码转为 Token 序列
2. **归一化**：统一标识符、字面量
3. **指纹提取**：生成 k-gram 哈希
4. **相似度计算**：比较指纹重叠度

### 3.5 优缺点

| 优点 | 缺点 |
|-----|------|
| 忽略格式差异 | 无法处理语句重排 |
| 能处理变量重命名 | 对代码结构变化敏感 |
| 效率较高 | 无法识别语义等价 |

## 4. 基于语法的方法（AST）

### 4.1 核心思想

将代码解析为抽象语法树（AST），比较树的结构相似度。

### 4.2 AST 示例

```javascript
// 源代码
const add = (a, b) => a + b;

// AST 结构
VariableDeclaration
├── VariableDeclarator
│   ├── Identifier: "add"
│   └── ArrowFunctionExpression
│       ├── params: [Identifier: "a", Identifier: "b"]
│       └── body: BinaryExpression
│           ├── operator: "+"
│           ├── left: Identifier: "a"
│           └── right: Identifier: "b"
```

### 4.3 关键技术

#### 4.3.1 结构签名（Structural Signature）

```typescript
/**
 * 生成 AST 节点的结构签名
 * 忽略具体的标识符名称，只保留结构信息
 */
function generateSignature(node: ASTNode): string {
  switch (node.type) {
    case 'FunctionDeclaration':
      // 函数签名：FUNC(参数数量)
      return `FUNC(${node.params.length})`;
    
    case 'IfStatement':
      // 条件语句签名
      return `IF{${generateSignature(node.consequent)}}`;
    
    case 'CallExpression':
      // 函数调用签名：CALL(参数数量)
      return `CALL(${node.arguments.length})`;
    
    // ... 其他节点类型
  }
}
```

#### 4.3.3 子树哈希

```python
def hash_subtree(node):
    """
    计算子树的哈希值，用于快速比较
    """
    if node is None:
        return 0
    
    # 递归计算子节点哈希
    children_hash = tuple(hash_subtree(child) for child in node.children)
    
    # 组合节点类型和子节点哈希
    return hash((node.type, children_hash))
```

### 4.4 功能等价检测

#### 4.4.1 循环等价

```javascript
// 以下三种写法功能等价
// 1. for 循环
for (let i = 0; i < arr.length; i++) {
  console.log(arr[i]);
}

// 2. forEach
arr.forEach(item => console.log(item));

// 3. for...of
for (const item of arr) {
  console.log(item);
}
```

检测方法：将不同的循环形式归一化为统一的抽象表示。

#### 4.4.2 条件等价

```javascript
// 以下两种写法功能等价
// 1. if-else
if (condition) {
  return a;
} else {
  return b;
}

// 2. 三元表达式
return condition ? a : b;
```

### 4.5 优缺点

| 优点 | 缺点 |
|-----|------|
| 忽略格式和命名差异 | 需要语言特定的解析器 |
| 能检测结构相似 | 计算复杂度较高 |
| 可识别部分语义等价 | 无法处理完全不同的实现 |

## 5. 基于语义的方法（深度学习）

### 5.1 核心思想

使用深度学习模型将代码编码为语义向量，通过向量相似度判断代码相似性。

### 5.2 代码表示学习

#### 5.2.1 Code2Vec

将代码路径（AST 中的路径）编码为向量：

```
代码 → AST → 提取路径 → 路径嵌入 → 聚合 → 代码向量
```

#### 5.2.2 CodeBERT / GraphCodeBERT

基于 Transformer 的预训练模型：

```python
from transformers import AutoModel, AutoTokenizer

# 加载 GraphCodeBERT
tokenizer = AutoTokenizer.from_pretrained("microsoft/graphcodebert-base")
model = AutoModel.from_pretrained("microsoft/graphcodebert-base")

def get_code_embedding(code: str):
    """
    获取代码的语义向量
    """
    inputs = tokenizer(code, return_tensors="pt", truncation=True, max_length=512)
    outputs = model(**inputs)
    
    # 使用 [CLS] token 的输出作为代码表示
    embedding = outputs.last_hidden_state[:, 0, :]
    return embedding
```

### 5.3 GraphCodeBERT 详解

#### 5.3.1 数据流图（Data Flow Graph）

GraphCodeBERT 的核心创新是引入数据流图：

```python
# 源代码
x = 1
y = x + 2
z = x * y

# 数据流图 (变量 → 依赖的变量)
# x → []        # x 没有依赖
# y → [x]       # y 依赖 x
# z → [x, y]    # z 依赖 x 和 y
```

#### 5.3.2 模型输入格式

```
[CLS] 代码Token序列 [SEP] 数据流变量序列 [SEP]
```

#### 5.3.3 相似度计算

```python
from sklearn.metrics.pairwise import cosine_similarity

def calculate_similarity(code1: str, code2: str) -> float:
    """
    使用 GraphCodeBERT 计算代码相似度
    """
    emb1 = get_code_embedding(code1)
    emb2 = get_code_embedding(code2)
    
    # 余弦相似度
    similarity = cosine_similarity(
        emb1.detach().numpy(),
        emb2.detach().numpy()
    )[0][0]
    
    return similarity
```

### 5.4 其他深度学习方法

| 方法 | 特点 | 适用场景 |
|-----|------|---------|
| **ASTNN** | 基于 AST 的 RNN | 代码分类 |
| **code2seq** | Seq2Seq + Attention | 代码摘要 |
| **InferCode** | 自监督学习 | 代码检索 |
| **UniXcoder** | 统一编码器 | 多任务 |

### 5.5 优缺点

| 优点 | 缺点 |
|-----|------|
| 能捕获深层语义 | 需要大量计算资源 |
| 可检测 Type-4 克隆 | 模型是黑盒，难以解释 |
| 跨语言能力 | 需要预训练数据 |

## 6. 混合方法

### 6.1 为什么需要混合方法

单一方法各有局限：

- **文本方法**：快但不准
- **AST 方法**：准但慢，且需要解析成功
- **深度学习**：强但慢，且难以解释

### 6.2 本项目的混合架构

```
┌─────────────────────────────────────────────────────┐
│                     用户界面                         │
└─────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────┐
│              前端 AST 分析（Babel）                   │
│  - 快速响应                                          │
│  - 结构签名比较                                      │
│  - 识别变量重命名、格式差异                          │
└─────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────┐
│           后端 GraphCodeBERT 语义分析                │
│  - 深层语义相似度                                    │
│  - 作为 AST 分析的补充验证                          │
└─────────────────────────────────────────────────────┘
```

### 6.3 分层检测策略

```python
def hybrid_similarity(code1: str, code2: str) -> dict:
    """
    混合相似度检测
    """
    result = {}
    
    # 第一层：快速文本检查

    result['text_similarity'] = text_sim
    
    if text_sim > 0.99:
        result['conclusion'] = '几乎完全相同'

    # 第二层：AST 结构分析

    result['semantic_similarity'] = semantic_sim

    return result
```

## 7. 实践应用场景

### 7.1 代码抄袭检测

**场景**：学术作业、编程竞赛

**关键点**：
- 需要处理各种混淆手段（重命名、重排序、插入无用代码）

**关键点**：

- 本项目的主要目标

**场景**：代码库检索、相似代码推荐

**关键点**：
